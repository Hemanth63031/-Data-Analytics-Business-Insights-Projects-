# 🛒 Retail Sales Analysis – SQL Project

## 📌 Project Overview
This project analyzes retail sales data using SQL to extract business insights such as customer behavior, product performance, revenue trends, and shipping status.

The goal is to simulate real-world data analysis tasks performed in business and operations teams.

---

## 📊 Dataset Description

The project contains 3 relational tables:

### 1. Customers Table
- customer_id
- first_name
- last_name
- country

### 2. Orders Table
- order_id
- customer_id
- item
- amount

### 3. Shippings Table
- shipping_id
- customer
- status (Delivered / Pending)

---

## 🎯 Objectives

- Analyze customer purchasing behavior
- Identify top-performing products
- Calculate total revenue and sales trends
- Understand shipping status distribution
- Detect repeat and high-value customers

---

## 🛠 Tools Used

- SQL (MySQL / SQL Server compatible)
- Relational database concepts
- Aggregations & Joins

---

## 📌 Key Business Questions Solved

- Total spending per customer
- Top 2 customers by revenue
- Highest & lowest selling products
- Revenue by country
- Customers with no orders
- Shipping status analysis
- Repeat customers identification

---

## 📈 Key Insights

- A small group of customers contributes majority of revenue
- Certain products generate significantly higher sales
- Some customers place multiple repeat orders
- Not all customers have shipping records
- Country-wise sales distribution shows clear leaders

---

## 🚀 How to Use

1. Import dataset into SQL database
2. Run queries from `sql_queries.sql`
3. Analyze outputs and derive insights
